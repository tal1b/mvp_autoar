User Stories
Story 1

As an administrator, I want to create an employee account, so I can manage workforce data.

Acceptance Criteria:

System accepts unique employee identifiers.

Employee record is stored in the database.

Employee appears in the employee list.

Story 2

As an employee, I want to log in to the system, so I can access my personal dashboard.

Acceptance Criteria:

Login requires valid credentials.

User is redirected to the dashboard after login.

Unauthorized access is blocked.

Story 3

As an administrator, I want to assign tasks, so employees know their responsibilities.

Acceptance Criteria:

Tasks can be created and assigned.

Assigned tasks are visible to employees.

Task data persists after reload.

Story 4

As an administrator, I want to assign work shifts, so schedules are organized.

Acceptance Criteria:

Shifts can be added and modified.

Employees can view their shifts.

Shift data is stored correctly.

Story 5

As an employee, I want to view my salary information, so I understand my compensation.

Acceptance Criteria:

Salary information is visible only to the employee.

Data is accurate and up to date.
